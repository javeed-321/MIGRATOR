## Mask Data in Databricks

Data masking can be done at the organization level to prevent the exposure of sensitive data. Admin users can enable data masking in Databricks and hide the PSI data (including custom field and extended field data) of customers during the export of data from Databricks. Once the [ETL](https://docs.capillarytech.com/docs/terms-and-abbreviations) is done, the data gets masked.

Perform the following steps to enable PII data masking:

1. From the Razer UI home page, navigate to **PII configs**.
2. For the brands you want to enable data masking, click **Configure**.

   <Image src="https://files.readme.io/d30b549-a3b39d1-Configure_2.png" width="1920" height="1080" alt="" />
3. Click **Enable**.

<Image src="https://files.readme.io/5281118-Enable_PII_masking.png" width="1920" height="1080" alt="" />

After enabling, when you export data, the PSI fields are masked.

<Image src="https://files.readme.io/91079d8-Masked_PII_data.png" width="1920" height="1080" alt="" />

4. To pause data masking, click **Pause** and to disable data masking, click **Disable**.

<Image src="https://files.readme.io/c0c1f1a-Disable.png" width="1920" height="1080" alt="" />

<br />

When [PSI data masking is enabled for a brand](https://docs.capillarytech.com/docs/classify-potentially-sensitive-information-psi), PSI fields including customer profile custom and extended fields, are masked in the Databricks. In the Analytics database (read_api tables), they are displayed as a string of asterisks ("*").

Create a Jira ticket for the product support team to pause or disable masking.

<Callout kind="info" title="Note">

To access unmasked PSI data, jobs must be run on admin_read_api tables, which are restricted to authorized Databricks users.

</Callout>
